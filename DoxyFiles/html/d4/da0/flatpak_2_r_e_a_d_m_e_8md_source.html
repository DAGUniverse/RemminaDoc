Remmina - The GTK+ Remote Desktop Client  1.2.0
Remmina is a remote desktop client written in GTK+, aiming to be useful for system administrators and travellers, who need to work with lots of remote computers in front of either large monitors or tiny netbooks. Remmina supports multiple network protocols in an integrated and consistent user interface. Currently RDP, VNC, NX, XDMCP and SSH are supported.

4 1. Install `flatpak` and `flatpak-builder` ([instructions]). Remmina
5  Flatpak manifest requires `flatpak-builder` >= 0.9.1.
6 
7 [instructions]: http://flatpak.org/getting.html
8 
9 2. Enable GNOME Flatpak repository:
10 
11  flatpak --user remote-add --if-not-exists --from gnome https://sdk.gnome.org/gnome.flatpakrepo
12 
13 3. Install GNOME runtime (have a look to the file `org.remmina.Remmina.json`
14  to get the required version, e.g, `"runtime-version": "3.26"`):
15 
16  flatpak --user install gnome org.gnome.Platform//3.26
17  flatpak --user install gnome org.gnome.Sdk//3.26
18 
19 4. Build Remmina:
20 
21  flatpak-builder --repo=repo/ appdir/ org.remmina.Remmina.json
22 
23  Remmina will be built in `appdir/` folder and the result will be exported
24  to a local Flatpak repository in `repo/` folder.
25 
26 5. Enable the local repository:
27 
28  flatpak --user remote-add --no-gpg-verify --if-not-exists my-repo repo/
29 
30 6. Install Remmina from your repository:
31 
32  flatpak --user install my-repo org.remmina.Remmina
33 
34 7. Launch Remmina
35 
36  flatpak run org.remmina.Remmina

38 Limitations
39 -----------
40 
41 Several Remmina features are not enabled in Flatpak build:
42 
43 * SPICE USB redirection uses a small suid wrapper
44  (`spice-client-glib-usb-acl-helper`) which is inhibited by Flatpak
45  `bubblewrap` sandboxing. Therefore, this feature is not enabled.
46 
47 * Telepathy DBus activation would require some more files exported outside of
48  Flatpak. Hence, Telepathy plugin is not compiled.
49 
50 * File transfers of some plugins (SFTP, SPICE drag and drop...) are limited to
51  the files located in the user home directory (see below).
52 
53 Security considerations
54 -----------------------
55 
56 * Remmina's Flatpak sandbox is configured to give access to the user home
57  directory. You can share more folders or remove them by using the command
58  `flatpak override --[no]filesystem=<folder>`.
59 
60 * Because Xephyr version `1.17.0` and later don't work fine with `GtkSocket`
61  (see downstream and upstream bugs), Remmina's Flatpak bundles an out of date
62  version of Xephyr (`1.16.4`) for the XDMCP plugin.
