3 # Remmina: The GTK+ Remote Desktop Client
4 
5 Initially developed by [Vic Lee](https://github.com/llyzs)
6 
7 ## Description
8 **Remmina** is a remote desktop client written in GTK+, aiming to be useful for
9 system administrators and travellers, who need to work with lots of remote
10 computers in front of either large monitors or tiny netbooks. Remmina supports
11 multiple network protocols in an integrated and consistent user interface.
12 Currently RDP, VNC, SPICE, NX, XDMCP and SSH are supported.
13 
14 Remmina is released in separated source packages:
15 * "remmina", the main GTK+ application
16 * "remmina-plugins", a set of plugins
17 
18 Remmina is free and open-source software, released under GNU GPL license.

20 ## Installation
21 
22 ### Binary distributions
23 Usually remmina is included in your linux distribution or in an external repository.
24 Do not ask for distribution packages or precompiled binaries here.
25 This is a development site.

27 ### Debian ###
28 
29 Remmina is not available on the default Debian 9 (Stretch) repositories. It can be installed from the Backports repository.
30 
31 The [Debian Backports](https://backports.debian.org/Instructions/) repository must be enabled to install it, see [this blog post](https://www.remmina.org/wp/debian-the-boys-are-backport-in-town/) for more information.
32 
33 To install Remmina from Debian Backports, just copy and paste the following lines on a terminal window:
34 
35 ```
36 echo 'deb http://ftp.debian.org/debian stretch-backports main' | sudo tee --append /etc/apt/sources.list.d/stretch-backports.list >> /dev/null
37 sudo apt update
38 sudo apt install -t stretch-backports remmina remmina-plugin-rdp remmina-plugin-secret libfreerdp-plugins-standard
39 ```

41 ### Ubuntu
42 
43 #### Using Snap Package (also for other [supported distros](https://snapcraft.io/docs/core/install))
44 
45 You can install the last release from the Ubuntu Software center, looking for `remmina`, otherwise you can install it from terminal with:
46 
47 ```sh
48 sudo snap install remmina
49 ```
50 
51 If you want to install latest git revision of remmina, you can use it from the `edge` channel:
52 
53 ```sh
54 sudo snap install remmina --edge
55 ```
56 
57 Or update the current installed version with the selected channel:
58 
59 ```sh
60 sudo snap refresh remmina --channel=edge # use --channel=stable otherwise
61 ```
62 
63 To enable some advanced features such as `mount-control` (to manage mount positions), `avahi-observer` (to automatically look for local servers to connect to), `cups-control` (to manage printing), `password-manager-service` (to use gnome-keyring) you should run something like:
64 
65 ```sh
66 sudo snap connect remmina:avahi-observe :avahi-observe # servers discovery
67 sudo snap connect remmina:cups-control :cups-control # printing
68 sudo snap connect remmina:mount-observe :mount-observe # mount management
69 sudo snap connect remmina:password-manager-service :password-manager-service # gnome-keyring
70 ```
71 
72 Snap packages will be updated automatically and will include both latest `FreeRDP` git and latest `libssh 0.7` release (for better security).
73 
74 #### From PPA
75 
76 [Ubuntu ppa:remmina-ppa-team/remmina-next](https://launchpad.net/~remmina-ppa-team/+archive/ubuntu/remmina-next)
77 
78 To install it, just copy and paste the following three lines on a terminal window
79 ```sh
80 sudo apt-add-repository ppa:remmina-ppa-team/remmina-next
81 sudo apt-get update
82 sudo apt-get install remmina remmina-plugin-rdp libfreerdp-plugins-standard
83 ```
84 By default the RDP, SSH and SFTP plugins are installed. You can view a list of available plugins with `apt-cache search remmina-plugin`
85 
86 If you want to connect to more securely configured SSH servers on Ubuntu 16.04 and below, you have to upgrade libssh to 0.7.X. This can be achieved by adding the following PPA containing libssh 0.7.X by David Kedves and upgrading your packages:
87 ```sh
88 sudo add-apt-repository ppa:kedazo/libssh-0.7.x
89 sudo apt-get update
90 ```

101 ### Arch Linux based
102 
103 Install [remmina-git](https://aur.archlinux.org/packages/remmina-git) from [AUR](https://aur.archlinux.org/)
104 
105 There are also some external, not supported plugins provided by [Muflone](https://github.com/muflone) :
106 
107 * [remmina-plugin-exec](https://aur.archlinux.org/packages/remmina-plugin-exec/) A protocol plugin for Remmina to execute an external process.
108 * [remmina-plugin-folder](https://aur.archlinux.org/packages/remmina-plugin-folder/) A protocol plugin for Remmina to open a folder.
109 * [remmina-plugin-open](https://aur.archlinux.org/packages/remmina-plugin-open/) A protocol plugin for Remmina to open a document with its associated application.
110 * [remmina-plugin-rdesktop](https://aur.archlinux.org/packages/remmina-plugin-rdesktop/) A protocol plugin for Remmina to open a RDP connection with rdesktop.
111 * [remmina-plugin-teamviewer](https://aur.archlinux.org/packages/remmina-plugin-teamviewer/) A protocol plugin for Remmina to launch a TeamViewer connection.
112 * [remmina-plugin-ultravnc](https://aur.archlinux.org/packages/remmina-plugin-ultravnc/) A protocol plugin for Remmina to connect via VNC using UltraVNC viewer.
113 * [remmina-plugin-url](https://aur.archlinux.org/packages/remmina-plugin-url/) A protocol plugin for Remmina to open an URL in an external browser.
114 * [remmina-plugin-webkit](https://aur.archlinux.org/packages/remmina-plugin-webkit/) A protocol plugin for Remmina to launch a GTK+ Webkit browser.

121 ## Usage
122 
123 Just select Remmina from your application menu or execute remmina from the command line
124 
125 Remmina support also the following options:
126 
127 ```sh
128  -a, --about Show about dialog
129  -c, --connect=FILE Connect to a .remmina file
130  -e, --edit=FILE Edit a .remmina file
131  -n, --new Create a new connection profile
132  -p, --pref=PAGENR Show preferences dialog page
133  -x, --plugin=PLUGIN Execute the plugin
134  -q, --quit Quit the application
135  -s, --server=SERVER Use default server name
136  -t, --protocol=PROTOCOL Use default protocol
137  -i, --icon Start as tray icon
138  -v, --version Show the application's version
139  -V, --full-version Show the application's version, including the pulgin versions
140  --display=DISPLAY X display to use
141 ```
